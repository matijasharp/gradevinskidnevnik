<objective>
## Goal
Migrate diary photo uploads from base64-in-Postgres to Supabase Storage, then verify all advanced features work end-to-end.

## Purpose
Photos stored as base64 strings in the DB create oversized rows and will degrade performance at scale. Supabase Storage provides proper blob handling with public CDN URLs. All other advanced features (AI, weather, reminders, calendar, PDF, analytics) are implemented — they need verification against the live Supabase backend.

## Output
- Supabase Storage bucket `diary-photos` with RLS policies
- `uploadDiaryPhoto` function in data.ts that uploads to Storage and returns a public URL
- App.tsx photo flow updated to use Storage URLs instead of base64
- All advanced features verified working end-to-end
</objective>

    TO authenticated
    WITH CHECK (bucket_id = 'diary-photos');

    CREATE POLICY "photos are publicly readable"
    ON storage.objects FOR SELECT
    USING (bucket_id = 'diary-photos');

    ON storage.objects FOR DELETE
    TO authenticated

    **Step 2 — Add `uploadDiaryPhoto` to data.ts:**
    Add after `createDiaryPhoto`:
    ```ts
    export const uploadDiaryPhoto = async (
      file: File,
      orgId: string,
      entryId: string
    ): Promise<string> => {
      ensureSupabase();
      const ext = file.name.split('.').pop() ?? 'jpg';
      const path = `${orgId}/${entryId}/${crypto.randomUUID()}.${ext}`;
      const { error } = await supabase.storage
        .from('diary-photos')
        .upload(path, file, { upsert: false, contentType: file.type });
      if (error) throw error;
      const { data } = supabase.storage.from('diary-photos').getPublicUrl(path);
      return data.publicUrl;
    };
    ```

    Update `deleteDiaryPhoto` to also remove from Storage:
    - After deleting the DB row, extract the storage path from the URL
      (pattern: `.../object/public/diary-photos/[path]`)
    - Call `supabase.storage.from('diary-photos').remove([path])`
    - Swallow Storage delete errors (file may not be in Storage for older base64 entries)

    **Step 3 — Update App.tsx photo state and submit flow:**

    to `{ url: string; description: string; file?: File }[]`

    In `onDrop` callback, add `file` to each photo state entry:
    ```ts
    setPhotos(prev => [...prev, { url: compressed, description: '', file }]);
    ```

    In `handleCameraCapture`, same change — store the File alongside the base64 preview.

    In `handleSubmit`, before calling `createDiaryPhoto`, try to upload to Storage:

    let photoUrl = photo.url; // base64 fallback
    if (photo.file && entryId) {

      companyId: appUser.companyId,

      description: photo.description
    });
