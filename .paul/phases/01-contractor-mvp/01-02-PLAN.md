<objective>
## Goal
Migrate authentication and all core CRUD writes from Firebase to Supabase while preserving the prototype UI and mobile design.

## Purpose
Supabase must be the source of truth for auth and data mutations to avoid Firebase auth errors and align with the Supabase-first backend.

## Output
- Supabase auth flow with onboarding and session handling
- Supabase write operations for projects, diary entries, photos, and user management
- Invitation flow compatible with Supabase schema and RLS
</objective>

## AC-1: Supabase auth replaces Firebase auth
```gherkin
Given Supabase Google OAuth is configured and env vars are set
When a user signs in
Then the app uses Supabase auth, loads the profile/org, and no Firebase auth errors occur
```

## AC-2: Core CRUD writes use Supabase
```gherkin
Given an authenticated user
When they create or update projects, diary entries, photos, or users
Then the writes persist to Supabase and list views update without UI changes
```

## AC-3: Invitation flow works with Supabase
```gherkin
Given an admin invites a user by email
When the invited user signs in
Then the user is linked to the organization and gains appropriate access
```
