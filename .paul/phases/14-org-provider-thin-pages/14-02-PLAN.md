# Phase 14 Plan 02: Thin Page Components

**Extract 7 route-specific page components from AppContent, extract Google Calendar state into
a reusable hook, fix the selectedProject URL-sync issue via an inline ProjectDetailRoute,
and remove the extracted state from AppContent.**

## Context

After Plan 14-01, `AppContent` is ~856 lines. All org-scoped data is available via `useOrg()`.
The Routes block in AppContent still has 11 inline route elements — each passes props from
AppContent's local state directly into view components. This plan extracts the 7 simpler routes
into standalone page components. The 4 complex routes (ProjectDetail, NewEntry, EditEntry,
MasterWorkspace) are deferred to Plan 14-03.

## Problem Analysis

Three categories of AppContent state to remove in this plan:

**1. Google Calendar state** — `googleTokens`, `calendarEvents`, `calendarLoading`,
`fetchCalendarEvents`, `connectGoogleCalendar`, message listener. Used by DashboardView and
CalendarView. Also `addToCalendar` (used by DiaryEntryDetailModal, stays in AppContent until
14-03). Extract into `useGoogleCalendar` hook; AppContent calls it too for `addToCalendar`.

**2. selectedProject / isSharedProject** — Set on navigation from ProjectsView/DashboardView;
read by the `/projects/:projectId` Route element (`selectedProject ? <ProjectDetailView> : null`).
This null-on-direct-URL issue was deferred from Phase 12. Fix here: add `ProjectDetailRoute`
inline component inside AppContent that uses `useParams()` + `projects.find()` from useOrg
closure. Removes the need for `selectedProject` and `isSharedProject` state.
DiaryEntryDetailModal's `project` prop switches to `projects.find(p => p.id === selectedEntry?.projectId)`.

**3. showInviteModal** — Moves into UsersPage. Also fixes the 14-01 regression where
`inviteUser` no longer called `setShowInviteModal(false)` on success.

```typescript
// Hook signature (not the full implementation — implementor fills in logic)
export function useGoogleCalendar() {
  // State: googleTokens, calendarEvents, calendarLoading
  // Effects:
  //   - on appUser change: load tokens from appUser.googleTokens + fetchCalendarEvents
  //   - on user logout (user === null): clear tokens
  //   - window message listener for GOOGLE_AUTH_SUCCESS → setGoogleTokens + updateProfileTokens + fetchCalendarEvents
  // Functions:
  //   - fetchCalendarEvents(tokens) — POST /api/calendar/events
  //   - connectGoogleCalendar() — GET /api/auth/google/url → window.open
  //   - addToCalendar(entry: DiaryEntry, projectName: string) — POST /api/calendar/add
  //     returns Promise<boolean> (true = success); does NOT call fetchCalendarEvents internally
  //     (caller decides whether to refresh)
  return { googleTokens, calendarEvents, calendarLoading, connectGoogleCalendar, addToCalendar }
}
```

Imports needed: `useAuth`, `updateProfileTokens` from data, `DiaryEntry` type.

```typescript
export default function UsersPage() {
  const { appUser, company } = useAuth();
  const { companyUsers, pendingInvitations, inviteUser, updateRole, deleteUser, cancelInvitation } = useOrg();
  const [showInviteModal, setShowInviteModal] = useState(false);

  if (appUser?.role !== 'admin') return null;

  const handleInvite = async (email: string, name: string, role: 'admin' | 'worker') => {
    await inviteUser(email, name, role);
    setShowInviteModal(false); // fixes 14-01 regression
  };

  return (
    <>
      <UsersView
        users={companyUsers}
        pendingInvitations={pendingInvitations}
        onInviteUser={() => setShowInviteModal(true)}
        onUpdateRole={updateRole}
        onDeleteUser={deleteUser}
        onCancelInvitation={cancelInvitation}
        currentUser={appUser}
      />
      {showInviteModal && (
        <InviteUserModal
          onClose={() => setShowInviteModal(false)}
          onSubmit={handleInvite}
          company={company}
        />
      )}
    </>
  );
}
```

```typescript
export default function CompanySettingsPage() {
  const { appUser, company, setCompany } = useAuth();

  if (appUser?.role !== 'admin' || !company) return null;

  const handleUpdate = (data: any) => {
    setCompany({ ...company, ...data });
    updateOrganization(company.id, data).catch((error) =>
      handleFirestoreError(error, OperationType.UPDATE, `organizations/${company.id}`)
    );
  };

  return <CompanySettingsView company={company} onUpdate={handleUpdate} />;
}
```

### Task 3 — Update App.tsx

This is the integration step. Changes in order:

**3a. Add `ProjectDetailRoute` component (inside AppContent, uses closure)**

Defined as a function component inside AppContent (so it closes over AppContent state):

```typescript
// Inside AppContent, before the return statement:
const ProjectDetailRoute = () => {
  const { projectId } = useParams();
  const project = projects.find(p => p.id === projectId) ?? null;
  const isShared = sharedProjects.some(sp => sp.id === projectId);
  if (!project) return null;
  return (
    <ProjectDetailView
      project={project}
      entries={entries.filter(e => e.projectId === project.id).sort((a, b) => b.entryDate.localeCompare(a.entryDate))}
      onBack={() => navigate(ROUTES.PROJECTS)}
      onNewEntry={() => navigate(ROUTES.NEW_ENTRY)}
      onEntryClick={setSelectedEntry}
      onGeneratePDF={async () => await generateDiaryPdf(project, entries.filter(e => e.projectId === project.id), company)}
      onAddToCalendar={addToCalendar}  // addToCalendar from useGoogleCalendar
      onCompleteProject={() => completeProject(project.id)}
      onUpdatePhase={(phase) => updateProjectPhase(project.id, phase)}
      onDeleteProject={() => deleteProject(project.id)}
      hasCalendar={!!googleTokens}
      userRole={appUser?.role}
      appUser={appUser}
      company={company}
      readonly={isShared}
      companyUsers={companyUsers}
    />
  );
};
```
