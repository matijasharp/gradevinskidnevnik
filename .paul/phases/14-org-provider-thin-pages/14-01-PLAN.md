<objective>
## Goal
Extract all organization-scoped data subscriptions and actions from AppContent into a new OrganizationProvider context, exposing them via `useOrg()`.

## Purpose
AppContent currently mixes auth state, org data subscriptions, UI state, and routing JSX. Extracting org-scoped data (projects, entries, users, invitations, masterProjects) into OrganizationProvider is the first step to reducing AppContent to ~100 lines. This pattern mirrors the already-successful AuthProvider extraction (Phase 11).

## Output
- `src/app/providers/OrganizationProvider.tsx` — new context + provider
- `src/app/providers/index.ts` — updated to export useOrg/OrganizationProvider
- `src/main.tsx` — OrganizationProvider wraps AppRouter (inside AuthProvider)
- `src/App.tsx` — AppContent uses useOrg(); all extracted state/subscriptions/actions removed
</objective>

  <name>Task 1: Create OrganizationProvider.tsx</name>
  <files>app/src/app/providers/OrganizationProvider.tsx</files>
  <action>
    Create `src/app/providers/OrganizationProvider.tsx` using AuthProvider as the structural template.

    The provider calls `useAuth()` internally to get `appUser` and `company`, then manages all org-scoped subscriptions and data.

    **Context interface — OrgContextValue:**
    ```ts
    interface OrgContextValue {
      // Subscribed data
      projects: Project[];
      entries: DiaryEntry[];
      companyUsers: AppUser[];
      pendingInvitations: Invitation[];
      // Fetched data
      sharedProjects: Project[];
      masterProjects: MasterProject[];
      // Derived
      materialHistory: string[];
      materialUnits: Record<string, string>;
      // User/team actions
      inviteUser: (email: string, name: string, role: 'admin' | 'worker') => Promise<void>;
      cancelInvitation: (invitationId: string) => Promise<void>;
      updateRole: (userId: string, newRole: 'admin' | 'worker') => Promise<void>;
      deleteUser: (userId: string) => Promise<void>;
      // Project actions
      createProject: (data: any) => Promise<void>;
      completeProject: (projectId: string) => Promise<void>;
      updateProjectPhase: (projectId: string, phase: Project['phase']) => Promise<void>;
      deleteProject: (projectId: string) => Promise<void>;
    }
    ```

    **Subscriptions to port from AppContent (preserve exact logic):**
    - `subscribeProjects(appUser.companyId, setProjects, errorHandler)` — depends on appUser
    - `subscribeDiaryEntries(appUser.companyId, setEntries, errorHandler)` — depends on appUser
    - `subscribeCompanyUsers(...)` + `subscribePendingInvitations(...)` — only when appUser.role === 'admin'
    - `fetchSharedProjects(appUser.id, appUser.companyId)` + `fetchMasterProjects(appUser.companyId)` — one-time fetch on appUser change

    **Derived state (port exactly from AppContent lines 261-270):**
    ```ts
    useEffect(() => {
      if (!entries.length) return;
      const allItems = entries.flatMap(e => e.lineItems || []);
      const historyMap = new Map<string, string>();
      allItems.forEach(item => {
        if (item.name && !historyMap.has(item.name)) historyMap.set(item.name, item.unit);
      });
      setMaterialHistory(Array.from(historyMap.keys()));
      setMaterialUnits(Object.fromEntries(historyMap));
    }, [entries]);
    ```

    **Actions to port (preserve exact logic, including alert/error behavior):**
    - `inviteUser` — calls createInvitation + POST /api/invite (needs `company` from useAuth())
    - `cancelInvitation` — calls cancelInvitationRecord
    - `updateRole` — calls updateProfileRole
    - `deleteUser` — calls removeOrganizationMember (include window.confirm)
    - `createProject` — calls createProjectRecord (needs navigate from useNavigate())
    - `completeProject` — calls updateProject (note: no longer mutates selectedProject — that stays in AppContent)
    - `updateProjectPhase` — calls updateProject (same note)
    - `deleteProject` — calls deleteProjectRecord + navigate(ROUTES.PROJECTS) (needs navigate)

    **Important:** `completeProject` and `updateProjectPhase` in AppContent also call `setSelectedProject(...)` to keep UI in sync. Since `selectedProject` stays in AppContent, those setSelectedProject calls must remain in AppContent's wrapper functions. The provider functions only do the DB update — AppContent's Route handlers will need to handle local state sync separately (or subscribe to the updated projects list, which the subscription already handles automatically). Remove the `setSelectedProject` mutation from the actions moved to OrganizationProvider — the Supabase subscription will update `projects` state automatically after the DB write.

    **Avoid:** Do not move googleTokens, calendarEvents, selectedProject, selectedEntry, editingEntry, or any modal state — those stay in AppContent.

    **navigate dependency:** `createProject` and `deleteProject` call `navigate()`. Use `useNavigate()` inside OrganizationProvider (valid since it's inside RouterProvider via AppRouter).
