  <action>
    Create migration file with:
    - ALTER TABLE organizations ADD COLUMN IF NOT EXISTS stripe_customer_id TEXT;
    - ALTER TABLE organizations ADD COLUMN IF NOT EXISTS subscription_status TEXT NOT NULL DEFAULT 'free';
    - Add CHECK constraint: subscription_status IN ('free', 'pro', 'cancelled', 'past_due')
    - Add RLS policy: org members can SELECT their own org's subscription_status
      (Use pattern from existing org policies — auth.uid() in org member query)
    - No UPDATE policy for subscription_status via client (webhook only)

    Avoid: modifying profiles table — billing is per-organization, not per-user.
  </action>

<task type="auto">
  <name>Task 2: Stripe endpoints in server.ts</name>
  <files>app/server.ts, app/.env</files>
  <action>
    1. Install Stripe SDK: add `import Stripe from 'stripe'` — use `npm install stripe` (note for user, not automated).
       At top of server.ts, initialize Stripe client:
       ```ts
       const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, { apiVersion: '2024-04-10' });
       ```

    2. Add Supabase admin client for webhook DB writes (needs service role key, bypasses RLS):
       ```ts
       import { createClient } from '@supabase/supabase-js';
       const supabaseAdmin = createClient(
         process.env.VITE_SUPABASE_URL!,
         process.env.SUPABASE_SERVICE_ROLE_KEY!
       );
       ```

    3. BEFORE `app.use(express.json())`, add raw body middleware for webhook route only:
       ```ts
       app.use('/api/stripe/webhook', express.raw({ type: 'application/json' }));
       ```
       Then keep the existing `app.use(express.json())` after.

    4. Add `POST /api/billing/create-checkout-session`:
       - Accepts { orgId, userEmail } from body
       - Creates Stripe customer if none exists (stripe.customers.create), stores in organizations.stripe_customer_id
       - Creates Checkout Session: mode 'subscription', price = STRIPE_PRICE_ID_PRO, success_url and cancel_url using APP_URL
       - Add metadata: { orgId } to session for webhook lookup
       - Returns { url: session.url }

    5. Add `POST /api/stripe/webhook`:
       - Get raw body: `req.body` (Buffer, already raw due to express.raw middleware above)
       - Verify signature: `stripe.webhooks.constructEvent(req.body, req.headers['stripe-signature'], STRIPE_WEBHOOK_SECRET)`
       - Handle event types:
         - `checkout.session.completed` → set subscription_status = 'pro', store stripe_customer_id from session.customer
         - `customer.subscription.deleted` → set subscription_status = 'cancelled'
         - `invoice.payment_failed` → set subscription_status = 'past_due'
       - All DB updates via supabaseAdmin, match org by session.metadata.orgId
       - Return { received: true } on success; 400 on invalid signature

    6. Update app/.env — add documented env var keys (with placeholder values):
       ```
       # Stripe
       STRIPE_SECRET_KEY=sk_test_...
       STRIPE_PRICE_ID_PRO=price_...
       STRIPE_WEBHOOK_SECRET=whsec_...
       SUPABASE_SERVICE_ROLE_KEY=...
       ```

    Avoid: putting STRIPE_SECRET_KEY or SUPABASE_SERVICE_ROLE_KEY in VITE_ prefixed vars (would expose to client bundle).
    Avoid: parsing JSON manually for the webhook — use raw Buffer and constructEvent only.
  </action>
  <verify>
    - npm run build passes (TypeScript compiles)
    - POST /api/billing/create-checkout-session with valid test keys returns a Stripe URL
    - Webhook endpoint returns 400 on invalid signature (testable with curl + wrong secret)
  </verify>
  <done>AC-2 and AC-3 satisfied: checkout session endpoint returns URL, webhook handler updates org subscription_status</done>
</task>
