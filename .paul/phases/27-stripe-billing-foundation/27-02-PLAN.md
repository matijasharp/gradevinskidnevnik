<objective>
## Goal
Add a BillingPage showing current plan and upgrade CTA, expose subscription_status in OrgContext, and gate project creation for free-tier orgs.

## Purpose
Gives users visibility into their subscription status and an upgrade path. Lays the enforcement foundation — one gate proves the pattern works before extending to more features.

## Output
- `subscription_status` exposed via OrgContext (fetched from organizations table)
- `/billing` route + `BillingPage` component
- Sidebar "Pretplata" nav link (below Postavke, above Super Admin)
- Free-tier gate on project creation: orgs on 'free' plan capped at 3 active projects
</objective>

## AC-1: Subscription Status in Context
```gherkin
Given a logged-in user with an organization
When OrganizationProvider mounts
Then useOrg().subscriptionStatus returns 'free' or 'pro' (fetched from organizations table)
```

## AC-2: Billing Page Visible
```gherkin
Given any authenticated user navigates to /billing
Then BillingPage renders showing current plan (Free or Pro), features list, and an "Nadogradi na Pro" button
```

## AC-3: Upgrade Flow Starts
```gherkin
Given a free-tier user clicks "Nadogradi na Pro" on BillingPage
When the button is clicked
Then a POST to /api/billing/create-checkout-session is made and the browser opens the returned Stripe Checkout URL in a new tab
```

## AC-4: Free Tier Project Gate
```gherkin
Given an organization on 'free' subscription_status with 3 or more active projects
When the user tries to create a new project
Then a blocking message "Besplatni plan omogućuje do 3 projekta. Nadogradi na Pro." is shown instead of the project form
```

       - Call `useOrg()` to get subscriptionStatus.
       - Show two-column layout (Free / Pro):

         - Pro column: "Nadogradi na Pro" button, lists pro features (unlimited projects, priority support, billing history)
       - "Nadogradi na Pro" button handler:
         - POST to `/api/billing/create-checkout-session` with `{ orgId: company.id, userEmail: appUser.email }`
         - On success: `window.open(url, '_blank')`
         - On error: show inline error message "Greška pri pokretanju naplate. Pokušajte ponovo."
       - If subscriptionStatus === 'pro': show "Aktivni Pro plan" badge instead of upgrade button; show Stripe customer ID last 6 chars as reference.
       - If subscriptionStatus === 'cancelled' or 'past_due': show warning banner "Pretplata je istekla ili čeka uplatu" + reactivate CTA.
       - Use existing shared/ui/Button, design tokens (--color-accent, --color-text-primary, etc.)

    2. Add to routeConfig.ts:
       ```ts
       BILLING: '/billing',
       ```

    3. Add to AppRouter.tsx — protected route for /billing:
       ```tsx
       <Route path={ROUTES.BILLING} element={<ProtectedRoute><BillingPage /></ProtectedRoute>} />
       ```

    4. Add to AppShell.tsx sidebar nav — "Pretplata" link with a CreditCard icon (use Lucide or inline SVG consistent with existing icons):
       - Position: below "Postavke" (company settings), above Super Admin link
       - Visible to all authenticated users (not gated by role — everyone can see billing)
       - Same active state pattern as other nav items (border-l-2 border-accent + bg-accent-subtle)

    Avoid: adding Stripe.js client-side SDK — server handles checkout session creation; client only opens the returned URL.
    Avoid: hardcoding the API URL — use relative `/api/billing/create-checkout-session` (server.ts serves it).
  </action>

    - Sidebar shows "Pretplata" link that navigates to /billing
  </verify>
