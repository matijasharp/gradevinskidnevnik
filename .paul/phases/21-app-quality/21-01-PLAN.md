<objective>
## Goal
Fix two calendar UI bugs: (1) clicking a project or diary entry in day view navigates to that project's detail page; (2) prev/next arrow buttons are visible and tappable on mobile.

## Purpose
Users currently cannot navigate from the calendar day view to a project — clicking does nothing. On mobile, the arrow buttons are not visible due to the custom Button component's `px-4 py-2` padding collapsing the 32px icon-button's content area, making the icon effectively invisible. Both are blocking usability issues in a core navigation surface.

## Output
- `CalendarView.tsx` — two targeted fixes; no other files changed
</objective>

<task type="auto">
  <name>Task 2: Fix mobile arrow button visibility</name>
  <files>app/src/features/calendar/components/CalendarView.tsx</files>
  <action>
    **Root cause:** The shared `Button` component always applies `px-4 py-2` padding. When `className="h-8 w-8"` sets the button to 32×32px, the 16px side padding collapses the content area to 0px, causing the icon to be clipped by the button's box model on mobile (where `overflow: hidden` on an ancestor may clip overflowing content).

    **Fix:** Replace the three arrow-area Buttons with raw `<button>` elements that have explicit icon-button classes. Do NOT modify the Button component — this is a local fix for the calendar nav only.

    Replace the entire `<div className="flex gap-1">` block (lines 157-167) with:

    ```tsx
    <div className="flex items-center gap-1">
      <button
        onClick={prev}
        className="flex items-center justify-center h-8 w-8 rounded-md text-zinc-600 hover:bg-zinc-100 active:scale-95 transition-all"
        aria-label="Prethodno"
      >
        <ChevronRight className="rotate-180" size={18} />
      </button>
      <button
        onClick={() => setCurrentDate(new Date())}
        className="flex items-center justify-center px-2 h-8 text-[10px] md:text-xs font-bold rounded-md text-zinc-600 hover:bg-zinc-100 active:scale-95 transition-all"
      >
        Danas
      </button>
      <button
        onClick={next}
        className="flex items-center justify-center h-8 w-8 rounded-md text-zinc-600 hover:bg-zinc-100 active:scale-95 transition-all"
        aria-label="Sljedeće"
      >
        <ChevronRight size={18} />
      </button>
    </div>
    ```

    This gives each button a guaranteed 32×32px clickable area with `flex items-center justify-center` ensuring the icon renders visibly in the center, no padding collapsing the box.
  </action>
