<objective>
## Goal
Fix mobile photo capture (remove camera-only lock, enable multi-select), add diary entry photos to project PDF exports, and rewrite the Izvještaji PDF to fix Croatian character encoding and add rich content (data tables, manual bar chart, project breakdown).

## Purpose
Field workers use mobile exclusively — broken camera capture and single-photo-only limits daily diary use. PDF exports are the primary deliverable for clients and supervisors; the current Izvještaji PDF is bare-bones and the Croatian character bug (čćžšđ showing as garbled) makes it unprofessional.

## Output
- Camera button on NewEntryView opens native OS picker (camera + gallery) with multi-select
- Diary project PDF includes photo thumbnails after each entry
- Izvještaji PDF exports with correct Croatian characters, project breakdown table, materials table, and a drawn bar chart of hours per day
</objective>
