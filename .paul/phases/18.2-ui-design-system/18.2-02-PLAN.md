<objective>
## Goal
Sweep all hardcoded `#6366f1` / `#3b82f6` hex fallbacks in feature files to `var(--color-accent)`, update shared UI component corners to `rounded` (4px), migrate StatusBadge to design token colors, and remove the `--color-primary` legacy token from index.css.

## Purpose
Plan 18.2-01 defined the tokens. This plan applies them. After this plan, no component references the old accent hex values — every `brandColor` fallback and accent usage flows through the CSS token system.

## Output
- Shared components: sharp corners (`rounded` instead of `rounded-xl`/`rounded-2xl`); StatusBadge uses token colors
- Feature files: `brandColor` fallbacks → `var(--color-accent)` (hex for PDF/color-picker exceptions)
- CalendarView heading: `text-primary` → `text-text-primary`
- index.css: `--color-primary` legacy entry removed
</objective>
