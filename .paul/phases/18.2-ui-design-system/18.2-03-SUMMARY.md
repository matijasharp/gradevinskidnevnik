## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| `forEach` → `for...of` in generateDiaryPdf | `await` inside a sync `forEach` callback is a compile error (esbuild/vite) | Required for async photo loading; semantically identical |
| `generateDiaryPdf` called directly in ProjectDetailView | `onGeneratePDF` prop doesn't have access to `projectPhotos` state | Cleaner — view owns its own PDF generation; prop still accepted on component signature |
| CSS var check before hexToRgb | `company.brandColor` may be `'var(--color-accent)'` after Plan 02 token sweep | Prevents jsPDF crash on color calls; falls back to `#3ab9e3` |

### Auto-fixed Issues

**1. Async/await in forEach callback**
- **Found during:** Task 2 qualify (build run)
- **Issue:** `await new Promise<void>(...)` inside `projectEntries.forEach(...)` callback — esbuild error: "await can only be used inside an async function"
- **Fix:** Converted `projectEntries.forEach((entry) => { ... })` to `for (const entry of projectEntries) { ... }`
- **Files:** `app/src/integrations/pdf/generateDiaryPdf.ts`
- **Verification:** `npm run build` exits 0
