<objective>
## Goal
Decompose `src/lib/data.ts` into 10 domain query modules under `src/integrations/supabase/queries/`, create a barrel index, and convert data.ts into a thin re-export.

## Purpose
data.ts is 1427 lines mixing 11 unrelated domains. Splitting it into domain modules makes it easier to import only what's needed per feature and paves the way for per-feature hooks in Plan 13-02. Zero behavior change — this is a mechanical move.

## Output
- `src/integrations/supabase/queries/` — 10 domain files with functions extracted from data.ts
- `src/integrations/supabase/index.ts` — barrel re-exporting all functions + types
- `src/lib/data.ts` — converted to `export * from '../integrations/supabase'`
</objective>
