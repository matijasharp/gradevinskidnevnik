Introduce the master project entity — a cross-org umbrella project that groups discipline subprojects from multiple contractor organizations — and build the Master Workspace tab with a list view and detail view.

  <name>Task 1: DB migration — master_projects + master_project_organizations + RLS</name>
  <files>supabase/migrations/20260328_07_master_workspace.sql</files>
  <action>
    Create two tables and their RLS policies.

    **master_projects table:**
    - id uuid PRIMARY KEY DEFAULT gen_random_uuid()
    - name text NOT NULL
    - description text
    - location text
    - status text NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'completed', 'archived'))
    - owner_organization_id uuid REFERENCES public.organizations(id) ON DELETE CASCADE
    - created_by uuid REFERENCES auth.users(id)
    - created_at timestamptz DEFAULT now()

    **master_project_organizations table:**
    - id uuid PRIMARY KEY DEFAULT gen_random_uuid()
    - master_project_id uuid REFERENCES public.master_projects(id) ON DELETE CASCADE
    - organization_id uuid REFERENCES public.organizations(id) ON DELETE CASCADE
    - discipline text NOT NULL CHECK (discipline IN ('electro', 'water', 'klima'))
    - role text NOT NULL DEFAULT 'contributor' CHECK (role IN ('lead', 'contributor', 'viewer'))
    - linked_project_id uuid REFERENCES public.projects(id) ON DELETE SET NULL
    - created_at timestamptz DEFAULT now()
    - UNIQUE (master_project_id, organization_id)

    **Indexes:**
    - master_project_organizations(master_project_id)
    - master_project_organizations(organization_id)

    **RLS — enable on both tables, then add policies:**

    master_projects:
    - SELECT: auth.uid() IS NOT NULL AND (
        owner_organization_id IN (SELECT organization_id FROM public.profiles WHERE id = auth.uid())
        OR id IN (SELECT master_project_id FROM public.master_project_organizations
                  WHERE organization_id IN (SELECT organization_id FROM public.profiles WHERE id = auth.uid()))
      )
    - INSERT: auth.uid() IS NOT NULL
    - UPDATE: auth.uid() IS NOT NULL AND owner_organization_id IN (SELECT organization_id FROM public.profiles WHERE id = auth.uid())
    - DELETE: same as UPDATE

    master_project_organizations:
    - SELECT: auth.uid() IS NOT NULL AND master_project_id IN (
        SELECT id FROM public.master_projects WHERE
          owner_organization_id IN (SELECT organization_id FROM public.profiles WHERE id = auth.uid())
          OR id IN (SELECT master_project_id FROM public.master_project_organizations AS inner_mpo
                    WHERE inner_mpo.organization_id IN (SELECT organization_id FROM public.profiles WHERE id = auth.uid()))
      )
    - INSERT: auth.uid() IS NOT NULL AND master_project_id IN (
        SELECT id FROM public.master_projects WHERE owner_organization_id IN (SELECT organization_id FROM public.profiles WHERE id = auth.uid())
      )
    - DELETE: same as INSERT

    Avoid: using project_members for master project membership — that table is for contractor-level cross-org sharing; keep concerns separate
  </action>

    ```
      role: 'lead' | 'contributor' | 'viewer';
      linkedProjectId?: string;

      createdAt: string;
    }
    ```

    **Private mappers:**
    ```
    const mapMasterProject = (row: any): MasterProject => ({ ... })
    const mapMasterProjectOrganization = (row: any): MasterProjectOrganization => ({ ... })
    ```

    **Functions to add:**

    createMasterProject(params: { name: string; description?: string; location?: string; ownerOrganizationId: string; createdBy: string }): Promise<MasterProject>
    - Insert into master_projects
    - Then insert owner org into master_project_organizations with role='lead', discipline fetched from organizations.discipline for ownerOrganizationId
    - Return created MasterProject

    fetchMasterProjects(organizationId: string): Promise<MasterProject[]>
    - Select from master_projects where owner_organization_id = organizationId OR id IN (SELECT master_project_id FROM master_project_organizations WHERE organization_id = organizationId)
    - Order by created_at desc

    fetchMasterProjectOrganizations(masterProjectId: string): Promise<MasterProjectOrganization[]>
    - Select from master_project_organizations with LEFT JOIN organizations(name) and LEFT JOIN projects(project_name) on linked_project_id
    - Where master_project_id = masterProjectId
    - Use Supabase select syntax: 'id, master_project_id, organization_id, discipline, role, linked_project_id, created_at, organizations(name), projects(project_name)'

    linkOrganizationToMasterProject(params: { masterProjectId: string; organizationId: string; discipline: 'electro' | 'water' | 'klima'; role?: 'lead' | 'contributor' | 'viewer'; linkedProjectId?: string }): Promise<void>
    - Upsert into master_project_organizations with onConflict: 'master_project_id,organization_id'

    updateMasterProject(id: string, updates: { name?: string; description?: string; location?: string; status?: string }): Promise<void>
    - Update master_projects where id = id

    Each function: call ensureSupabase() at start, throw error on Supabase error.
    Avoid: modifying any existing functions or interfaces in data.ts
