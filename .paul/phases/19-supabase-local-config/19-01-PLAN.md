<objective>
## Goal
Create a reproducible local dev environment: Supabase CLI config, auto-generated TypeScript types, and a seed SQL file with representative dev data.

## Purpose
Developers can run `supabase start` to spin up a local Postgres+Auth+Storage instance that mirrors production schema, then `supabase db reset` to seed it with realistic test data — no manual setup, no shared staging environment needed.

## Output
- `supabase/config.toml` — Supabase CLI config for local dev
- `app/src/integrations/supabase/database.types.ts` — TypeScript types generated from live schema
- `supabase/seed.sql` — seed SQL with 2 orgs, 2 users, 1 client, 2 projects, 3 diary entries, 1 master project
</objective>
