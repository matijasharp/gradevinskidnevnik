<objective>
## Goal
Add a `discipline` field to organizations and replace all hardcoded electrical phase/work-type arrays with discipline-driven config — so water and klima contractors see their own terminology without any code changes.

## Purpose
The app currently has electro phases and work types hardcoded in three separate places in App.tsx. This blocks onboarding of any non-electro discipline. Extracting the config to a single module and wiring it to `organizations.discipline` unlocks all three disciplines with no duplication.

## Output
- Supabase migration: `discipline` column on `organizations`
- New file: `app/src/lib/disciplineConfig.ts` — per-discipline phase and work-type maps + helpers
- Updated `app/src/lib/data.ts` — Company interface, mapOrganization, updateOrganization, widened phase types
- Updated `app/src/App.tsx` — hardcoded arrays replaced, discipline picker added to CompanySettingsView
</objective>
