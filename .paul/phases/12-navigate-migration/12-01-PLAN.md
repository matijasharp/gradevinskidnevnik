<objective>
## Goal
Replace all `setView()` / `view` state navigation with `useNavigate()` + `<Routes>`, activate AppRouter as the RouterProvider, and remove the temporary BrowserRouter from main.tsx.

## Purpose
Phase 11 wired BrowserRouter and AuthProvider as a bridge. Phase 12 activates real URL-based navigation: the back button works, direct URL access works, and navigation calls use ROUTES constants instead of view strings.

## Output
- `App.tsx`: `view` useState removed; `<Routes>` replaces view-conditional rendering; all `setView()` replaced with `navigate()`
- `AppShell.tsx`: `view`/`setView` props replaced with `useNavigate()` / `useLocation()`
- `AppRouter.tsx`: catchall route renders `<App />`; placeholder stubs removed
- `ProtectedRoute.tsx` / `GuestRoute.tsx`: wired with `useAuth()` guards
- `main.tsx`: BrowserRouter removed; `<AppRouter />` renders the RouterProvider
</objective>

    **ProtectedRoute.tsx** â€” wire useAuth() guard:
    ```tsx
    import { Navigate } from 'react-router-dom';
    import { useAuth } from '../providers';
    import { ROUTES } from './routeConfig';

    export default function ProtectedRoute({ children }: { children: React.ReactNode }) {
      const { loading, user } = useAuth();
      if (loading) return null;            // AuthProvider handles loading state
      if (!user) return <Navigate to={ROUTES.DASHBOARD} replace />;  // AppContent shows LoginView at '/'
      return <>{children}</>;
    }
    ```
    Note: redirect to ROUTES.DASHBOARD (not /login) because AppContent shows LoginView when !user â€” there is no separate /login route yet (that's Phase 14).

    **GuestRoute.tsx** â€” wire useAuth() guard:
    ```tsx
    import { Navigate } from 'react-router-dom';
    import { useAuth } from '../providers';

    export default function GuestRoute({ children }: { children: React.ReactNode }) {
      const { loading, user } = useAuth();
      if (loading) return null;
      if (user) return <Navigate to={ROUTES.DASHBOARD} replace />;
    ```
