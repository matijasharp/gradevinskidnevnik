<objective>
## Goal
Fix splash screen animation replaying on slow auth loads, shorten animation duration, and update GCP OAuth consent screen branding.

## Purpose
Two remaining Phase 25 items: (1) the splash animation replays when Supabase auth resolves slowly because `onComplete` is a new function reference on each LoginView render — GSAP's `useEffect([onComplete])` re-fires and restarts the animation; (2) the GCP OAuth consent screen still shows the generic/Supabase app name instead of branded "Građevinski Dnevnik Online".

## Output
- SplashScreen.tsx: reduced animation durations (total ~0.8s instead of ~1.3s)
- LoginView.tsx: `useCallback` on `onComplete` handler to stabilize the reference
- GCP OAuth consent screen updated with branded app name and support email (manual step)
</objective>

<task type="auto">
  <name>Task 1: Fix animation replay + shorten durations</name>
  <files>app/src/features/auth/components/LoginView.tsx, app/src/features/auth/components/SplashScreen.tsx</files>
  <action>
    **LoginView.tsx:**
    - Add `useCallback` import from React
    - Wrap the `onComplete` handler in `useCallback`:
      ```tsx
      const handleSplashComplete = useCallback(() => setSplashDone(true), []);
      ```
    - Pass `handleSplashComplete` instead of the inline arrow to SplashScreen:
      ```tsx
      return <SplashScreen onComplete={handleSplashComplete} />;
      ```
    - This stabilizes the reference across re-renders — GSAP's `useEffect([onComplete])` won't re-fire when `loading` flips.

    **SplashScreen.tsx — reduce durations:**
    - Logo fade-in: `duration: 0.6` → `duration: 0.4`
    - Text fade-in: `duration: 0.5` → `duration: 0.3`
    - Fade-out delay: `delay: 0.1` → `delay: 0.05`
    - Fade-out duration: `duration: 0.4` → `duration: 0.25`
    - New total: logo (0.4s) + text starts at -0.2s offset → ends at 0.2+0.3=0.5s + hold 0.05s + fadeout 0.25s = ~0.8s total

    Do NOT change the `-=0.3` overlap offset in `tl.fromTo` for text — adjust to `-=0.2` to keep the stagger proportional to new durations.

    Avoid: changing the animation sequence, easing, or layout. Only timing values change.
  </action>
  <verify>
    `npm run build` in `app/` passes with no errors.
    Visually: open app in browser, observe splash plays once and completes in under 1 second.
  </verify>
  <done>AC-1 satisfied: stable onComplete ref prevents replay. AC-2 satisfied: total duration ≤ 0.9s.</done>
