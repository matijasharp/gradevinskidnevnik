<objective>
## Goal
Fix the broken `poveži Google Calendar` OAuth flow and clean up the Supabase Auth URL configuration so the app uses the production domain correctly in all OAuth contexts.

## Purpose
Both OAuth flows are broken or show incorrect branding post-launch:
- Google Calendar connect flow fails because the production redirect URI (`https://elektro.gradevinskidnevnik.online/auth/google/callback`) is not registered in Google Cloud Console
- Supabase Auth Site URL is not set to the production domain, so redirect allowlisting is incomplete

## Output
- Supabase Auth Site URL = `https://elektro.gradevinskidnevnik.online`, production URL in redirect allowlist
- Google Cloud Console: production redirect URI registered, consent screen authorized domain added
- Railway: `APP_URL` env var confirmed set to production URL
- `.env.example` updated with `APP_URL` documented
- Google Calendar OAuth flow works end-to-end on production
</objective>

<task type="checkpoint:human-action" gate="blocking">
  <action>Update Supabase Auth URL Configuration for production domain</action>
  <instructions>
    1. Open Supabase Dashboard → your project → Authentication → URL Configuration
    2. Set **Site URL** to: `https://elektro.gradevinskidnevnik.online`
    3. Under **Redirect URLs**, add: `https://elektro.gradevinskidnevnik.online/**`
       (the `/**` wildcard covers all deep-link redirects)
    4. Save changes

    This ensures Supabase Auth redirects users back to the production app after any OAuth login (Google Sign-in via Supabase), not to localhost or a stale URL.
  </instructions>
  <verification>Settings saved — no error shown in Dashboard</verification>
  <resume-signal>Type "done" when Supabase URL config is saved</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Register production redirect URI in Google Cloud Console + verify APP_URL on Railway</action>
  <instructions>
    **Part A — Google Cloud Console:**
    1. Open https://console.cloud.google.com → your project → APIs & Services → Credentials
    2. Find the OAuth 2.0 Client ID used for Google Calendar
    3. Under **Authorized redirect URIs**, add:
       `https://elektro.gradevinskidnevnik.online/auth/google/callback`
       (Keep the existing `http://localhost:3000/auth/google/callback` for local dev)
    4. Under **OAuth consent screen** → check **Authorized domains** — add `gradevinskidnevnik.online` if not present
    5. Save

    **Part B — Railway environment variables:**
    1. Open Railway dashboard → your service → Variables
    2. Confirm `APP_URL` is set to: `https://elektro.gradevinskidnevnik.online`
       (If missing or wrong, set it now and redeploy)

    The Express server builds the redirect URI as `${APP_URL}/auth/google/callback` (server.ts:24).
    If `APP_URL` is localhost, the OAuth popup gets a redirect_uri mismatch error.
  </instructions>
  <verification>
    After saving GCP credentials, call `https://elektro.gradevinskidnevnik.online/api/auth/google/url`
    and confirm the returned URL contains `redirect_uri=https%3A%2F%2Felektro.gradevinskidnevnik.online%2Fauth%2Fgoogle%2Fcallback`
  </verification>
  <resume-signal>Type "done" when GCP redirect URI is registered and APP_URL is confirmed on Railway</resume-signal>
</task>
