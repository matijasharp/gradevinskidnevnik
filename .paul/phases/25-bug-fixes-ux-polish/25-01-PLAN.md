<objective>
## Goal
Fix 5 production UX bugs: mobile camera opens gallery instead of camera, project not pre-selected when creating a new entry from a project, scroll not reset on page navigation, splash screen too slow, and activity tab empty for cross-org project members.

## Purpose
All five are post-launch friction points reported in the v1.3 scope. Fixing them together brings the app to a polished, expected-behaviour baseline.

## Output
- `NewEntryView.tsx` — `capture="environment"` on the Slikaj file input
- `NewEntryPage.tsx` — reads `projectId` from route state and passes `initialProject`
- `ProjectDetailPage.tsx` — passes `project.id` as route state when navigating to new entry
- `SplashScreen.tsx` — animation capped (delay 0.5→0.2s, total duration ≤1.5s)
- `AppRouter.tsx` — `ScrollToTop` component resets scroll on every route change
- New Supabase migration — adds activity_log SELECT policy for cross-org project members
</objective>

<task type="auto">
  <name>Task 2: Project pre-selection + ScrollToTop on navigation</name>
  <files>app/src/features/projects/components/ProjectDetailPage.tsx, app/src/features/diary/components/NewEntryPage.tsx, app/src/app/router/AppRouter.tsx</files>
  <action>
    **Fix 1 — ProjectDetailPage.tsx:**
    The `onNewEntry` handler currently navigates to `ROUTES.NEW_ENTRY` with no state.
    Pass the project ID as route state:
    ```tsx
    onNewEntry={() => navigate(ROUTES.NEW_ENTRY, { state: { projectId: project.id } })}
    ```

    **Fix 2 — NewEntryPage.tsx:**
    Add `useLocation` import and read the `projectId` from route state.
    Use it to find the matching project and pass as `initialProject`:
    ```tsx
    import { useNavigate, useLocation } from 'react-router-dom';
    // inside component:
    const location = useLocation();
    const initialProjectId = (location.state as any)?.projectId;
    const initialProject = initialProjectId
      ? projects.find(p => p.id === initialProjectId) ?? null
      : null;
    // pass to NewEntryView:
    initialProject={initialProject}
    ```

    **Fix 3 — AppRouter.tsx:**
    Add a `ScrollToTop` component that calls `window.scrollTo(0, 0)` whenever the pathname changes.
    Place it inside the router, before the Routes:
    ```tsx
    import { useEffect } from 'react';
    import { useLocation } from 'react-router-dom';

    function ScrollToTop() {
      const { pathname } = useLocation();
      useEffect(() => { window.scrollTo(0, 0); }, [pathname]);
      return null;
    }
    ```
    Then render `<ScrollToTop />` as the first child inside the RouterProvider's root element (inside AppRouter's route structure, not outside RouterProvider).
  </action>

<task type="auto">
  <name>Task 3: Fix activity_log RLS for cross-org project members</name>
  <files>supabase/migrations/20260403_16_activity_log_project_member_select.sql</files>
  <action>
    The current `activity_log` SELECT policy gates on `organization_id in (select org from organization_members)`.
    Cross-org project members are not in the project owner's org, so they get empty results with no error.

    Create migration file `supabase/migrations/20260403_16_activity_log_project_member_select.sql`:
    ```sql
    -- Allow cross-org project members to read activity for their projects
    -- The existing "Org members can read activity_log" policy covers own-org users.
    -- This policy adds visibility for invited cross-org members by project_id.
    create policy "Project members can read activity_log"
      on public.activity_log for select
      using (
        project_id in (select get_my_project_ids())
      );
    ```

    Then apply this migration to the database via MCP.
  </action>

    3. Run `npm run build` — must pass
  </verify>
