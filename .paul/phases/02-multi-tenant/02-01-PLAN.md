<objective>
## Goal
Add project-level membership management and cross-org project invitations so that users from different organizations can be invited to collaborate on specific projects.

## Purpose
Phase 1 built org-scoped access — every org member sees every org project. Phase 2 adds the next layer: project-level membership, where an admin controls which members (including external users) have access to a specific project. This is the foundation for the "invited participation in master projects" feature.

## Output
- `project_invitations` DB table + RLS policies
- data.ts functions: project member CRUD + project invitation flow
- App.tsx: project members tab (add/remove/role), cross-org invite acceptance on login, Shared Projects section
</objective>

    Create migration file `supabase/migrations/20260328_04_project_invitations.sql`:
    ```sql
    create table if not exists public.project_invitations (
      id uuid primary key default gen_random_uuid(),
      project_id uuid references public.projects(id) on delete cascade,
      email text not null,
      name text,
      role text not null default 'viewer',  -- lead | contributor | viewer
      created_by uuid references auth.users(id),
      created_at timestamptz default now(),
      unique (project_id, email)
    );
    create index if not exists project_invitations_project_idx on public.project_invitations(project_id);
    create index if not exists project_invitations_email_idx on public.project_invitations(email);
    ```

    Create policies file `supabase/policies/20260328_04_project_invitations_policies.sql`:
    - Enable RLS on project_invitations
    - SELECT: project_id in get_my_org_project_ids() OR lower(email) = lower(auth.email())
    - INSERT: project_id in get_my_org_project_ids()
    - DELETE: project_id in get_my_org_project_ids() OR lower(email) = lower(auth.email())

    Also add an INSERT policy for project_members that allows a user to insert themselves
    when a project_invitation exists for their email:
    ```sql
    create policy project_members_self_insert on public.project_members
      for insert
      with check (
        user_id = auth.uid()
        and exists (
          select 1 from public.project_invitations pi
          where pi.project_id = project_id
            and lower(pi.email) = lower(auth.email())
        )
      );
    ```
    (Add this to the policies file as a drop/recreate of project_members_access or as an additional policy.)

    Apply both files to Supabase via the SQL editor (same process as prior migrations).

<task type="auto">
  <name>Task 2: Add project membership functions to data.ts</name>
  <files>app/src/lib/data.ts</files>
  <action>
    Add the following exports to data.ts (after the existing invitation functions, around line 850+):

    **Types** (add to existing type section at top of file):
    ```ts
    export interface ProjectMember {
      id: string;
      projectId: string;
      userId: string;
      role: 'lead' | 'contributor' | 'viewer';
      invitedBy?: string;
      createdAt: string;
      // joined from profiles
      name?: string;
      email?: string;
    }

    export interface ProjectInvitation {
      id: string;
      projectId: string;
      email: string;
      name?: string;
      role: 'lead' | 'contributor' | 'viewer';
      createdBy?: string;
      createdAt: string;
    }
    ```

    **Functions**:

    `fetchProjectMembers(projectId: string): Promise<ProjectMember[]>`
    - Query project_members WHERE project_id = projectId
    - Left join profiles on user_id to get name and email
    - Map to ProjectMember[]

    `addProjectMember(params: { projectId: string; userId: string; role: 'lead' | 'contributor' | 'viewer'; invitedBy: string }): Promise<void>`
    - Insert into project_members (project_id, user_id, role, invited_by)
    - On conflict (project_id, user_id) do update role

    `removeProjectMember(params: { projectId: string; userId: string }): Promise<void>`
    - Delete from project_members WHERE project_id = projectId AND user_id = userId

    `updateProjectMemberRole(params: { projectId: string; userId: string; role: 'lead' | 'contributor' | 'viewer' }): Promise<void>`
    - Update project_members SET role = role WHERE project_id = projectId AND user_id = userId

    `fetchProjectInvitations(projectId: string): Promise<ProjectInvitation[]>`
    - Query project_invitations WHERE project_id = projectId
    - Map to ProjectInvitation[]

    `createProjectInvitation(params: { projectId: string; email: string; name?: string; role: 'lead' | 'contributor' | 'viewer'; createdBy: string }): Promise<ProjectInvitation>`
    - Insert into project_invitations
    - Return mapped record

    `cancelProjectInvitation(invitationId: string): Promise<void>`
    - Delete from project_invitations WHERE id = invitationId

    `fetchProjectInvitationByEmail(email: string): Promise<ProjectInvitation | null>`
    - Query project_invitations WHERE lower(email) = lower(input email)
    - Returns first match or null (used on login to auto-accept)

    `acceptProjectInvitation(params: { invitation: ProjectInvitation; userId: string }): Promise<void>`
    - Insert into project_members (project_id, user_id, role, invited_by) — on conflict update role
    - Delete from project_invitations WHERE id = invitation.id

    `fetchSharedProjects(userId: string): Promise<Project[]>`
    - Query projects WHERE id IN (SELECT project_id FROM project_members WHERE user_id = userId)
    - AND organization_id NOT IN (get_my_organization_ids for this user — i.e., filter out own-org projects)
    - Practical implementation: fetch all project_members rows for userId, get project_ids,
      then fetch those projects, filter out ones where organization_id matches user's org
    - Map to Project[]

    Avoid: importing anything new — use existing `supabase` client from supabase.ts.
    Avoid: changing existing function signatures.
